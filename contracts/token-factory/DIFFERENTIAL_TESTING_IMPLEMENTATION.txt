# Differential Testing Implementation - Issue #373

## Overview
Implemented comprehensive differential testing system to validate vesting and supply math by comparing contract outputs against a high-precision reference implementation.

## Components Created

### 1. Reference Arithmetic Engine (differential_engine.rs)
- **DifferentialVestingSchedule**: High-precision vesting calculation
- **DifferentialEngine**: Manages multiple vesting schedules
- **SupplyTracker**: Tracks supply conservation with balance verification

### 2. Comprehensive Differential Tests (comprehensive_differential_tests.rs)
- **test_differential_vesting_linear_progression**: Validates linear vesting across time
- **test_differential_vesting_boundary_timestamps**: Tests edge cases (u64::MAX, zero values)
- **test_differential_vesting_large_amounts**: Tests with i128::MAX values
- **test_differential_supply_conservation**: Verifies mint/burn supply invariants

### 3. Property-Based Differential Tests (differential_proptest.rs)
- **prop_vesting_never_exceeds_total**: Vested amount ≤ total amount
- **prop_vesting_monotonic**: Vesting increases over time
- **prop_supply_conservation**: Supply = minted - burned
- **prop_rounding_consistency**: Contract matches reference rounding
- **prop_claim_idempotence**: Claiming twice at same time yields zero

## Test Coverage

### Randomized Schedules
- Total amounts: 1 to 1,000,000,000
- Start times: 1,000 to 100,000
- Cliff durations: 0 to 1,000
- Vesting durations: 100 to 10,000
- Current times: 1,000 to 200,000

### Edge Cases
- Boundary timestamps (0, u64::MAX - 10000)
- Large values (i128::MAX / 2, i128::MAX / 10)
- Rounding precision (365-day vesting)
- Integer saturation edges

### Mismatch Diagnostics
Each failure produces:
- Test name
- Full input vector (all parameters)
- Contract output
- Reference output
- Difference magnitude

## Running Tests

```bash
# Run all differential tests
./run_differential_tests.sh

# Run specific test suite
cargo test comprehensive_differential_tests -- --nocapture
cargo test differential_proptest -- --nocapture
cargo test differential_test -- --nocapture
```

## Implementation Notes

- Reference engine uses identical algorithm to contract for exact comparison
- All arithmetic uses checked operations to detect overflow/underflow
- Supply tracker verifies: total_supply = total_minted - total_burned
- Supply tracker verifies: sum(balances) = total_supply
- Property tests run with configurable iteration count (default: 256)
- Comprehensive tests use deterministic test vectors
- Mismatch reports include actionable diagnostics for debugging
